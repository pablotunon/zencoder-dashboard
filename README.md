# AgentHub Analytics

Organizational analytics dashboard for monitoring AI agent usage, cost, and performance across engineering teams.

## Architecture

```
                        ┌──────────┐
                        │  nginx   │ :8080
                        │  proxy   │
                        └────┬─────┘
               ┌─────────┬──┴──────────┐
               │         │             │
        ┌──────▼──┐ ┌────▼─────┐ ┌─────▼──────┐
        │ Frontend│ │Analytics │ │ Ingestion  │
        │  React  │ │   API    │ │   (Rust)   │
        │   SPA   │ │ (FastAPI)│ │   Axum     │
        └─────────┘ └──┬──┬───┘ └──┬─────────┘
                       │  │        │
          ┌────────────┘  │   ┌────┘
          │               │   │
     ┌────▼────┐  ┌───────▼─┐ │  ┌────────────────┐
     │PostgreSQL│  │ClickHouse│ ├─▶│  Redis Streams  │
     │ metadata │  │analytics │ │  └───────┬────────┘
     └─────────┘  └─────────┘  │          │
                               │  ┌───────▼────────┐

                    │          │  └────────────────┘
              ┌─────▼─────┐   │

              └───────────┘
```

### Services

| Service | Language | Purpose |
|---------|----------|---------|
| **Frontend** | TypeScript (React + Vite) | Dashboard SPA with Tremor charts, shadcn/ui shell |
| **Analytics API** | Python (FastAPI) | BFF serving pre-aggregated data from ClickHouse/PostgreSQL/Redis |
| **Ingestion** | Rust (Axum) | High-throughput event intake, publishes to Redis Streams |
| **Aggregation Worker** | Python | Consumes events from Redis, inserts into ClickHouse, computes rollups |
| **Simulator** | TypeScript (Node.js) | Seeds PostgreSQL with demo org data, generates 90 days of events |

### Infrastructure

| Component | Purpose |
|-----------|---------|
| **PostgreSQL 16** | Organization, team, user, and project metadata |
| **ClickHouse 24** | Time-series event storage and aggregated rollup tables |
| **Redis 7** | Event bus (Streams), real-time counters, API response cache |
| **nginx** | Reverse proxy routing `/api/*`, `/ingest/*`, and `/` to services |

## Quick Start

```bash
# Clone and start the full stack
docker compose up --build -d

# Wait for all services to initialize (roughly 60-90 seconds for
# the simulator to seed data and backfill 90 days of events)

# Open the dashboard
open http://localhost:8080

# Check service health
curl http://localhost:8080/api/health
curl http://localhost:8080/ingest/health

# View API documentation (auto-generated by FastAPI)
open http://localhost:8080/api/docs
```

## Dashboard Pages

- **Overview** — KPI cards (total runs, active users, cost, success rate), usage trend chart, team breakdown table
- **Usage & Adoption** — Adoption rate, active users trend (DAU/WAU/MAU), agent type breakdown, top users, project breakdown
- **Cost & Efficiency** — Cost trend, breakdown by team/agent type/project, cost per run, token usage, budget tracking
- **Performance & Reliability** — Success/failure rate trend, latency percentiles (p50/p95/p99), error breakdown, queue wait times

## API Endpoints

All `/api/metrics/*` endpoints accept query parameters:
- `period`: `7d`, `30d`, `90d` (default: `30d`)
- `teams`: comma-separated team slugs
- `projects`: comma-separated project IDs
- `agent_types`: comma-separated agent types

| Method | Path | Description |
|--------|------|-------------|
| GET | `/api/health` | Service health with dependency status |
| GET | `/api/orgs/current` | Current organization with teams and projects |
| GET | `/api/metrics/overview` | KPI cards, usage trend, team breakdown |
| GET | `/api/metrics/usage` | Adoption rate, active users, agent type/project breakdown |
| GET | `/api/metrics/cost` | Cost trends, breakdown, token usage, budget |
| GET | `/api/metrics/performance` | Success rate, latency, error breakdown |
| POST | `/ingest/events` | Ingest event batch (1-100 events) |
| GET | `/ingest/health` | Ingestion service health |

## Development

All commands run inside Docker containers per project constraints.

```bash
# Rebuild a single service
docker compose up --build -d <service_name>

# Run tests
docker compose exec simulator npm run test
docker compose exec ingestion cargo test
docker compose exec aggregation-worker pytest
docker compose exec analytics-api pytest

# Linting
docker compose exec simulator npm run lint
docker compose exec ingestion cargo fmt
docker compose exec ingestion cargo clippy

# Run E2E smoke tests (requires stack to be running)
./tests/e2e/run.sh

# Stop everything
docker compose down

# Full reset (wipes volumes)
docker compose down -v && docker compose up --build -d
```

## Tech Stack Rationale

- **Rust (Ingestion)** — Zero-cost abstractions for high-throughput event processing; Axum provides async-first HTTP with compile-time guarantees
- **Python (API + Worker)** — FastAPI for rapid BFF development with auto-generated OpenAPI docs; ClickHouse/PostgreSQL client libraries are mature in Python
- **TypeScript (Frontend + Simulator)** — React + Vite for fast iteration; Tremor for analytics-grade charts out of the box
- **ClickHouse** — Column-oriented analytics engine purpose-built for time-series aggregation at scale
- **Redis Streams** — Lightweight event bus with consumer groups, avoiding the operational complexity of Kafka for a demo-scale system
