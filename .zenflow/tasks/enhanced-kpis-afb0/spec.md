# Technical Specification: Enhanced KPI Panels

## Difficulty: Easy–Medium

Frontend-only change. No backend or API modifications. Three files modified. The main complexity is in the sparkline rendering and layout of the enhanced KPI widget.

---

## Technical Context

- **Language**: TypeScript (React 19)
- **UI framework**: Tailwind CSS 4, Headless UI
- **Charting**: recharts 2.15 (already a dependency)
- **Data layer**: React Query (@tanstack/react-query)
- **Testing**: Vitest + Testing Library
- **Build**: Vite 6

---

## Problem

The current `KpiWidget` component (`WidgetRenderer.tsx:656-683`) is minimal — it shows only:
- A large aggregate value
- A `+X% vs prev period` change indicator

The Copilot Metrics Dashboard (reference) enriches its stat cards with descriptions, tooltips, and contextual info. Our KPI panels can be significantly richer using data **already returned by the API** (the full timeseries is in every KPI response but unused).

---

## Implementation Approach

### What the enhanced KPI widget displays

1. **Value + Sparkline** — The large metric value (left) with a small ~48px-tall area sparkline (right), using the existing `data[]` timeseries array. Partial buckets excluded from the sparkline.

2. **Change % + Previous value** — Existing change % indicator, plus a "was $X" line showing the previous period's value. Previous value is derived client-side: `prevValue = value / (1 + change_pct / 100)`.

3. **Description** — A short static sentence explaining what the metric means, from a new `description` field on `MetricMeta`.

4. **Period high/low** — `min`/`max` over non-partial `data[].value` entries, displayed as "Low $X · High $Y" in a subtle footer.

### Layout

```
┌─────────────────────────────────────┐
│  $12,345.00              ╱╲__╱──   │  value (left) + sparkline (right)
│  +8.3% vs prev period              │  change % (green/red)
│  was $11,400.02                    │  previous value (muted)
│                                     │
│  Total spend across all agent       │  description (muted, text-xs)
│  runs in the selected period        │
│  ── ── ── ── ── ── ── ── ── ──    │  subtle divider
│  Low $234.50  ·  High $2,100.00    │  period extremes (muted, text-xs)
└─────────────────────────────────────┘
```

## Source Code Changes

### 1. `frontend/src/types/widget.ts`

Add `description` to the `MetricMeta` interface:

```typescript
export interface MetricMeta {
  key: MetricKey;
  label: string;
  description: string;  // NEW — human-readable explanation of the metric
  category: MetricCategory;
  defaultChartType: ChartType;
  compatibleChartTypes: ChartType[];
  format: ValueFormat;
  validBreakdowns: BreakdownDimension[];
  color: string;
}
```

### 2. `frontend/src/lib/widget-registry.ts`

Add `description` to each of the 14 metric entries:

| Metric | Description |
|--------|-------------|
| `run_count` | "Total agent executions in the selected period" |
| `active_users` | "Unique users who triggered at least one agent run" |
| `cost` | "Total spend across all agent runs" |
| `cost_per_run` | "Average cost for each agent execution" |
| `tokens_input` | "Total input tokens consumed by all runs" |
| `tokens_output` | "Total output tokens generated by all runs" |
| `success_rate` | "Percentage of runs that completed successfully" |
| `failure_rate` | "Percentage of runs that ended in failure" |
| `error_rate` | "Percentage of runs that produced a categorized error" |
| `latency_p50` | "Median execution time across all runs" |
| `latency_p95` | "95th percentile execution time" |
| `latency_p99` | "99th percentile execution time" |
| `queue_wait_avg` | "Average time runs spent waiting in queue" |
| `queue_wait_p95` | "95th percentile queue wait time" |

### 3. `frontend/src/components/widgets/WidgetRenderer.tsx`

Rewrite the `KpiWidget` function (~13 lines → ~60 lines):

- Accept the full `WidgetTimeseriesResponse` (already does) plus `metricMeta: MetricMeta`
- Render sparkline using recharts `ResponsiveContainer` + `AreaChart` + `Area` (no axes, no tooltip, no grid)
- Compute `prevValue` from `summary.value` and `summary.change_pct`
- Compute `min`/`max` from `data[]` filtering out `is_partial === true`
- Display description from `metricMeta.description`
- Pass `metricMeta` from `SingleChartDispatch` (which already has access to `METRIC_REGISTRY[metric]`)

---

## Data Model / API Changes

**None.** All data needed is already in the `WidgetTimeseriesResponse`:
- `summary.value` — current aggregate
- `summary.change_pct` — change vs previous period
- `data[]` — full timeseries with `{ timestamp, value, is_partial }`

The `MetricMeta` type gets a new `description` field (frontend-only type, not an API contract).

---

## Edge Cases

| Condition | Behavior |
|-----------|----------|
| `change_pct === null` | Hide "was X" line (no previous period data) |
| `data` array is empty | Hide sparkline and high/low line |
| All data points are `is_partial` | Hide sparkline and high/low line |
| Single data point (non-partial) | Show sparkline as single dot; high === low → show single value |
| `change_pct === 0` | Show "0.0% vs prev period" in neutral gray |
